# 사진전 (automatic-octo-gallery)

개인 사진전을 준비하고 여는 Next.js 갤러리 앱입니다. 사진 파일은 서버 로컬 경로에 그대로 두고, 앱은 그 경로를 인덱싱해 카드/썸네일 보기, 다중 선택, 콜라주 보기, 태그(가상 폴더) 관리, 태그 병합, 슬라이드쇼(6종 전환효과), 일괄 삭제(휴지통)/이동을 제공합니다.

## 시작하기

```bash
npm install
npx prisma migrate deploy   # SQLite DB(prisma/app.db) 생성
npm run dev
```

<http://localhost:3000> 에서 확인할 수 있습니다.

## 환경 변수

- `DATABASE_URL` — SQLite 파일 경로 (기본값 `.env`에 설정됨: `file:./prisma/app.db`)
- `GALLERY_EXPLORER_ROOT` — 사진 경로를 선택할 때 파일 탐색기가 넘어갈 수 없는 최상위 경계. 설정하지 않으면 서버 실행 계정의 홈 디렉토리가 기본값입니다.

## 주요 개념

- **관람 모드 / 큐레이터 모드**: 헤더의 토글 버튼으로 전환합니다. 관람 모드는 그리드·라이트박스·슬라이드쇼만 보이는 읽기 전용 화면이고, 큐레이터 모드에서만 태그 관리·메타데이터 편집·설정·일괄 액션이 노출됩니다.
- **태그 = 사진전**: 여러 폴더에 흩어진 사진을 태그로 묶어 하나의 사진전처럼 관리합니다. 두 태그를 병합하면 두 태그의 합집합을 담은 새 태그가 만들어지고, 기존 두 태그는 그대로 유지됩니다.
- **휴지통**: 삭제는 실제로는 사진 경로 하위의 `.trash` 폴더로 이동하는 소프트 삭제이며, 휴지통 페이지에서 원래 위치로 복구할 수 있습니다.
- **경로 샌드박싱**: 파일 탐색기와 모든 파일 작업(썸네일, 원본, 이동, 삭제/복구)은 서버 측에서 realpath 기반으로 경계를 검증해 `../` 이스케이프나 심볼릭 링크 우회를 차단합니다.
